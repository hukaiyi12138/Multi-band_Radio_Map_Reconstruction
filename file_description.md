model.py — PR_BTD_DULRTC with N_iter unrolled BTD-ADMM blocks. The ray-cast operator integrates the building map along TX→pixel rays, the free-space anchor (exp(-n·log_d - α·T_r)) gives a (0,1]-bounded path-loss prior, and SLFProximalBlock learns only the residual on top of the anchor with near-zero (std=1e-3) final-layer initialisation so training begins from the physics-only prediction. All ADMM subproblems (X, S_r, c_r, E, Z) have closed-form updates; the last block skips its prox since its P output never enters the loss.
util.py — Three datasets sharing a common 5-tuple return signature (D_obs, Ω, B, tx_pos, meta). For BART-LAB, mode='legacy' reads your existing 256×256 .mat files unchanged; mode='multi_crop' reads the raw maps and emits 4 reflection-padded crops per map (one centred on each of the 3 TXs + one on their centroid). For RadioMapSeer, it loads PNGs from gain/DPM/, png/buildings_complete/, png/antennas/ with synthetic random Ω. extract_tx_positions does dilation-based NMS on the TX heatmap to pick R peaks.
train.py — Adapted from your existing pipeline: Adam, cosine LR, gradient clipping, NaN guards, best-by-val-L1 checkpointing. --dataset {bartlab,radiomapseer,synthetic} switches between sources, --mode {legacy,multi_crop} picks the BART-LAB loading strategy.
test.py — Reports L1, RMSE, NMSE (dB), PSNR (dB), and the binary coverage outage rate at a chosen threshold. Optional matplotlib visualisations (GT / observed / estimate / error map per sample). Smoke run on the synthetic dataset reached 61 dB PSNR, confirming the architecture can recover BTD-structured ground truth.
BART-LAB practical notes
For your existing 256×256 .mat data, run with --mode legacy --omega-ratio 10 (or whichever OMEGA{N} folder you already use). If you have spare time later and want the ~4× sample boost, point --mode multi_crop at the raw BART-LAB directory — reflection-padded TX-centred crops are cleaner training signal than zero-padded full-map crops because they keep the building mask consistent with the physical environment (no phantom free-space regions). The legacy and multi-crop modes are independent; you can mix runs.
One quirk worth knowing: the closed-form c_r update can drive emitter PSDs to exactly 0 when fit is poor on early iterations (especially on random/noisy data). With a properly initialised model on real radio maps this is almost never an issue, but if you see metrics flatline mid-training and emitter PSDs collapsing, the fix is to swap torch.clamp(c_new, min=0.0) for F.softplus(c_new) in BTD_RPCA_Block.forward — it preserves non-negativity with smooth gradient. I left the hard clamp in place because it matches the BTD identifiability literature and worked on the synthetic benchmark.
